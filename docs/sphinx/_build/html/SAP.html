SAP.model module¶

This module contains functionalities of modeling and evaluation of LSTM models that are designed to estimate remaining useful life of turbofan engine data.

SAP.model.Evaluate_Model(Xtesting, Ytesting, engines, model, all_in_one, engines_to_check)¶

This function evaluate the trained model on the test data for specified engines

Parameters:
  • Xtesting (numpy array) – test input data
  • Ytesting (numpy array) – test label data
  • engines (numpy array or list) – array/list of engines that are collected in test data
  • model (Keras Model Sequential) – trained model
  • all_in_one (boolean (True/False)) – whether test data to be prepared separately for every engine or combined; separate option is better as it provides ease to understand the model evaluation results; combined option is not available
  • engines_to_check (list) – list of test engines that are tested on trained model
Returns:

tuple (prediction, real, engines_to_check)
SAP.model.LSTM_for_RUL(X, centering, normalization, dt, load_XYtraining_data, model_parameters, import_trained_model, plot_training_history)¶

This function first prepare the training data for modeling. For modeling, it accepts model architecture.

Parameters:
  • X (Pandas Dataframe) – Training dataframe; each row corresponds to an engine at specific cycle and its operational parameters and sensors measurements at that cycle
  • centering (boolean (True/False)) –
    • option to select whether input data to be min-max normalized and centered for better learning
  • normalization (boolean (True/False)) – option to select whether output label data to be normalized
  • dt (integer (e.g. 10,20,30 etc...)) – span of cycles as a single input to be learn by the model
  • load_XYtraining_data (boolean (True/False)) – option to select whether to load already prepared training data that are available only for dt = 10, 20, 30, 40, 50
  • model_parameters (dictionary) – neural network architecture parameters
  • import_trained_model (Keras Model Sequential) – Keras trained model
  • plot_training_history (boolean (True/False)) – whether to plot model training error per epoch
Returns:

tuple (training data dataframe, training input data, training label data, operational features, sensors features, cycle features, centering (True/False), normalization (True/False))
SAP.model.Prepare_TestData(X, testing_RUL, settings_min_max_mean, sensors_min_max_mean, time_of_event_max, centering, normalization, dt, all_in_one)¶

This function prepares the testing data for model evaluation.

Parameters:
  • X (Pandas Dataframe) – Test dataframe; each row corresponds to an engine at specific cycle and its operational parameters and sensors measurements at that cycle
  • testing_RUL (Pandas Dataframe) – remaining useful life for every engine (at the end of cycle) in the test data
  • settings_min_max_mean (dictionary) – operational settings features
  • sensors_min_max_mean (dictionary) – sensor meaturements features
  • time_of_event_max (float) – maximum value of cycle from any engine in training data
  • centering (boolean (True/False)) –
    • whether input data is min-max normalized and centered for better learning
  • normalization (boolean (True/False)) – whether output label data is normalized
  • dt (integer (e.g. 10,20,30 etc...)) – span of cycles as a single input to be learn by the model
  • all_in_one (boolean (True/False)) – whether test data to be prepared separately for every engine or combined; separate option is better as it provides ease to understand the model evaluation results
Returns:

tuple (Test input data, Test true label data, engines label for test input data, all_in_one to be used in any following functions)
